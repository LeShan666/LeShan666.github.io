禹州实训-宠物医院(前端)

  讲真，我对前端还是比较抵触的，因为我觉得这事是需要专业的人来做的(我们一点都不专业，前端课都是水课)。用户体验，界面设计，效果啥的，本来我就没审美……还想让我咋地！再说了，前端开发，目的是将用户体验提高到100分，但是这玩意给我的开发体验简直就是0分，没法让我感到”写得爽“。直到有一天我遇见了Angular，我靠前端还能这么开发？Service?依赖注入？MVC？这不是spring吗？还有typescript什么鬼，比原来那个辣鸡js好用多了，一看还是巨硬出得，巨硬终于干点人事了。

前端技术栈选择

Angular

What is Angular?
  Angular 是一个开发平台。它能帮你更轻松的构建 Web 应用。Angular 集声明式模板、依赖注入、端到端工具和一些最佳实践于一身，为你解决开发方面的各种挑战。Angular 为开发者提升构建 Web、手机或桌面应用的能力。

  上面设一段是Angular官网对这个框架的描述，上面说了这个框架写起来有一些像spring，因为Google走的就是Java技术栈而且从Stack Overflow的统计数据来看，也是Java与C#程序员使用着居多，很多人将其称之为后端专用的前端框架。Angular这是一个大而全的框架，并不像React和Vue那样还需要配合其他东西才能发挥作用。当然这个框架的设计模式是基于Google多年的Java编程经验的，响应式的应用也是基于微软对于操作系统中异步处理的经验总结。总之，我就喜欢这种架子大（健壮性好），规矩多（编程规范及约束）的框架。

TypeScript

  选这个是因为Angular本身就是使用TypeScript写的，在代码规范性上来说，很大一部分都要归功于TypeScript，因为他是强类型语言。

动态类型一时爽，代码重构火葬场。

  高手写出来的js代码确实很厉害，但是菜鸡写出来的js代码也是没有下限，并且要命的是每个人风格还不一样，团队合作能乱成一锅粥。ts就不一样了，强大的类型约束使得很多东西只能那么写，严格的规范也限制了很多的写法，导致所有人的风格都是一样的，这种情况下进行团队协作的效率极高。

  其实最早我是想学Vue的，毕竟国人写的，而且文档比较全，国内社区也活跃，据说还挺简单，只是Angular这种前端写法吸引我了，有时间可以稍微看看Vue。

项目准备

开启angular项目

  Angular是依靠node.js的，所以使用的时候要先安装node.js。有了node.js之后，首先要安装Angular CLI，这是Angular的脚手架，后面很多东西也要依靠他，然后在命令行输入。

npm install -g @angular/cli

国内安装比较慢的话可以使用阿里的cnpm，怎么装网上一大堆，这里就不多说了。之后就可以使用Angular的ng指令了。

ng new my-app

这个命令会创建一个名为my-app的Angular项目，中间会让选择，回车就完事了，然后在my-app目录下使用

ng serve –open

浏览器弹出欢迎界面就表示创建成功了。

引入Angular Material

  https://material.angular.cn/guide/getting-started 这里有全部的安装过程，我使用的是在项目目录下使用

ng add @angular/material
